✅ 修复内容：解决了 autoUpdate 模式下节点拖拽后回到原位的问题

🎯 测试说明

修复的问题：

  • 时序问题：修复了数据更新和状态重置的时序问题
  • 状态管理：确保数据更新在拖拽状态重置之前完成
  • 异步处理：使用 nextTick 确保 DOM 更新完成

测试步骤：

  1. 访问 AutoUpdate 示例页面
  2. 在"自动更新模式"部分拖拽节点到新位置
  3. 观察节点是否停留在新位置（而不是回到原位）
  4. 检查数据显示区域是否正确更新
  5. 同时测试"手动处理模式"确保没有破坏原有功能

🔍 修复详情

修复前的问题：

event.accept = () => {
  // ❌ 错误的顺序：先清理状态，后更新数据
  onDrop(event.originalEvent, event.dropNode)
  resetDragState()
  
  if (props.autoUpdate) {
    emit('update:value', updatedData) // 数据更新可能失效
  }
}

修复后的代码：

event.accept = () => {
  if (props.autoUpdate && !event.isCrossTree) {
    try {
      // ✅ 正确的顺序：先更新数据
      const updatedData = moveTreeNode(...)
      emit('update:value', updatedData)
      
      // ✅ 等待数据更新完成后再清理状态
      nextTick(() => {
        onDrop(event.originalEvent, event.dropNode)
        resetDragState()
      })
    } catch (error) {
      // ✅ 错误处理
      console.error('自动更新数据失败:', error)
      onDrop(event.originalEvent, event.dropNode)
      resetDragState()
    }
  } else {
    // ✅ 非自动更新模式保持原有逻辑
    onDrop(event.originalEvent, event.dropNode)
    resetDragState()
  }
}
